Smalljava

The First release of Smalljava is Available

Smalljava is a Java API for generating bytecodes in a natural, convenient way.

Smalljava is the first phase of the Moksa project. Once Smalljava is stable, work will begin on MoksaProlog, a 100% Java Prolog environment, which will become the basis of Moksa.

Smalljava is based on the excellent JavaClass library, written by Markus Dahm. You need the JavaClass library in order to run Smalljava. This library is available either from the JavaClass web site, or from this page.

Obtaining Smalljava

The following entities are available for download.

  • Smalljava JAR file - required - The compiled class files for Smalljava.
  • JavaClass JAR file - required - JavaClass library: required in order to run Smalljava.
  • Smalljava source - The source code for Smalljava (written entirely in Java).
  • The Moksa JavaDoc bundle - This is the Moksa JavaDoc documentation bundle. It has lots of extraneous documentation, to nonreleased packages from Moksa, but also includes the two packages which have been released: com.svincent.util and com.svincent.smalljava. You'll need this if you want to do any programming with Smalljava.

Requirements

As of this time, Smalljava requires a working installation of the JRE1.2 (or the Java 2.0 platform, if you believe in the new name). This requirement is due to my use, in particular, of the JDK1.2 collections API. It is probably possible to get it to run on earlier versions of the JDK, but I don't particularly want to do it.

Installing Smalljava

There are two very easy ways to install Smalljava.

Way 1

The first way is probably the easiest. Just copy smalljava.jar and javaclass.jar into your JRE standard extensions directory, and run away.

Way 2

The second way is sometimes better. Modify your CLASSPATH environment variable to include javaclass.jar and smalljava.jar, and that's it.

Checking your installation

To check to ensure that Smalljava is installed correctly, go to some empty directory, and type 

    java com.svincent.smalljava.SmallClass

This should create two files in the current directory: 'Person.java' and 'Person.class'. You can directly run 'Person.class' by typing 

    java Person

This will spew lots of loop outputs and "Hello World"'s to your screen, indicating that Smalljava is installed correctly
